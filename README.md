# Restart Journey Sim

An interactive iPhone walkthrough of the **Maximus UK Restart Scheme** customer journey, with a live AI adviser ("Sarah") running on **Amazon Bedrock AgentCore Runtime** and **Strands Agents SDK**.

> Demo only. Not affiliated with DWP or the real Maximus Restart programme.

Live demo: a CloudFront URL is printed by `deploy-web.py`. Sign-in is gated by Amazon Cognito so credentials are not exposed in the static bundle.

---

## What the demo shows

A long-term Universal Credit claimant (Daniel Okafor) is referred to Restart by DWP, walks through 12 customer touchpoints across SMS, push notifications, email, in-app cards, and a live AI-powered adviser chat.

The sim opens with a "Matrix-style" routing animation that decodes Daniel's top five customer attributes against the Maximus adviser pool and locks onto Priya Sharma at 98% compatibility, then lands on a pixel-faithful iPhone running the journey.

### The 12 stages

| # | Stage | Channel | What happens |
|---|-------|---------|--------------|
| 1 | Welcome and first contact | SMS | DWP referral; warm welcome text; cross-links to Adult Skills Fund and Access to Work |
| 2 | Diagnostic appointment booked | SMS | First face-to-face confirmed with Priya |
| 3 | Nudge: night before first meeting | Push | Checklist: ID, NI number, travel, mindset |
| 4 | Your action plan | Email | Personalised plan with weekly goals |
| 5 | CV and skills support | Push | CV ready to review + funded Level 2 course |
| 6 | Job match | Push | 92% match: Warehouse Team Member at Sundown Logistics |
| 7 | Interview and prep | SMS | Interview scheduled, travel costs covered |
| 8 | Nudge: night before interview | Push | Sleep, outfit, route, mindset checklist |
| 9 | Nudge: morning of interview | Push | 2-hour countdown: eat, leave, breathe, smile |
| 10 | Job offer | Email | Offer details, UC taper plain-English, cross-links to Access to Work and Adult Skills Fund |
| 11 | Nudge: night before first shift | Push | Boots, alarms, travel pass, bed by 10 |
| 12 | In-work support | Push | 6 months of weekly check-ins |

### Live adviser escalation

Three escalation surfaces, all backed by the same AgentCore agent:

- **"Talk to a live adviser" chip** in the SMS thread (escalates from bot to human-style)
- **Live Chat home-screen app** (iMessage-style blue thread with a presence indicator)
- **Push notification handoff** ("Sarah is ready")

Sarah is a Strands agent with four tools:

| Tool | What it does |
|------|--------------|
| `lookup_action_plan()` | Returns Daniel's current goal, weekly tasks, CV status, training, appointments |
| `book_appointment(date, slot, mode)` | Creates a new appointment with a confirmation code; persists in state |
| `find_jobs(keywords, postcode, limit)` | Ranks 5 mock local vacancies with match score and commute time |
| `request_callback(reason, urgency)` | Schedules a callback from a specialist (mental health, debt, housing) |

## Architecture

```
Browser (CloudFront / S3)
  │
  ├─ Cognito user-pool sign-in (USER_PASSWORD_AUTH) ──→ JWT
  │
  └─ InvokeAgentRuntime (Authorization: Bearer <JWT>)
       │
       ▼
     Bedrock AgentCore Runtime
       │  container image on ECR
       │  ARM64, built by AWS CodeBuild
       │
       └─→ Strands Agent (Sarah persona)
              ├─ system prompt (UK Restart adviser voice)
              ├─ Strands tools (book, find, lookup, callback)
              └─ Bedrock model (Amazon Nova Pro)
```

Why Nova Pro instead of Claude Haiku 4.5: account 348425780214 hasn't submitted the Anthropic use-case form yet, so Claude returns `ResourceNotFoundException`. Nova Pro supports tools and works immediately. Switch back by changing `MODEL_ID` in `agent/deploy.py` once the form is approved.

## Setup from scratch

Prereqs: AWS account with Bedrock model access (Nova Pro or Anthropic if you have the form approved), AWS CLI configured, Python 3.12+, Node 18+.

```bash
# 1. Clone
git clone https://github.com/636137/restart-journey-sim
cd restart-journey-sim

# 2. Create Python venv for the agent
python3 -m venv ~/.venvs/restart-adviser
source ~/.venvs/restart-adviser/bin/activate
pip install -r agent/requirements.txt boto3

# 3. Deploy the agent + Cognito to your account
#    Creates ECR repo, builds image via CodeBuild, creates AgentCore runtime, Cognito pool with JWT authorizer.
python3 agent/deploy.py
# Output is written to agent/deploy-output.json

# 4. Build the browser bundle
cd web
npm install
npm run build
cd ..

# 5. Deploy the site to S3 + CloudFront
python3 deploy-web.py
# Output is written to web-deploy-output.json
# Open the CloudFront URL it prints. Sign in with the credentials shown.
```

### Iterating

| Change | Redeploy with |
|--------|---------------|
| Frontend (HTML / CSS / JS in `index.html`) | `python3 deploy-web.py` |
| `agent-client.js` (Cognito / AgentCore wiring) | `cd web && npm run build && cd .. && python3 deploy-web.py` |
| Strands tools, agent prompt, AgentCore image | `python3 agent/deploy.py` (rebuilds the container) |
| Agent env vars only (e.g. swap model id) | Edit `agent/deploy.py` and rerun, or `aws bedrock-agentcore-control update-agent-runtime` directly |

### Local dev without AgentCore

```bash
# Terminal 1 - agent
~/restart-journey-sim/agent/run.sh        # FastAPI on :8000

# Terminal 2 - site
cd ~/restart-journey-sim
python3 -m http.server 8765               # http://localhost:8765
```

The Live Chat banner reads "Local dev mode" when the bundle can't reach AgentCore.

## Security notes

- Credentials are not baked into the static bundle. The page renders a sign-in overlay that authenticates against Cognito via `USER_PASSWORD_AUTH` and then talks to AgentCore as a Bearer JWT.
- The S3 bucket is private; CloudFront uses an Origin Access Control to fetch from it.
- AgentCore Runtime authenticates via the Cognito-pool discovery URL (`customJWTAuthorizer`). SigV4 access is not configured.
- To rotate the Admin password: `aws cognito-idp admin-set-user-password --user-pool-id <id> --username Admin --password <new> --permanent`
- To add more users: `aws cognito-idp admin-create-user --user-pool-id <id> --username <name> --temporary-password <pw>` then `admin-set-user-password ... --permanent`
- For PoCs only. Anyone signed in shares Sarah's session memory and can mutate `state.json`.
